Github

Share and collaborate your project

Create a Repository

  1. Go to GitHub.
  2. Create a new repository for your project by clicking on the New in the Repositories.
  3. Name the repository and select whether it should be Public or Private.
  4. Check the box for Add a README file.

Configure Git Account

Check Currently Active Git Account

View the account used by the current project (priority: project > global).

git config --show-origin user.name
git config --show-origin user.email

View global configuration.

git config --global --list | grep user

Configure Git Account Uniformly

Global settings (default for all projects).

git config --global user.name "Astra726"
git config --global user.email "XXX"

Project-level settings (current project only).

git config user.name "Astra726"
git config user.email "XXX"

Push Your Project to GitHub

On your local machine where the project exists:

# Navigate to Your Project Directory
cd /path/to/your/project

# Delete the local .git folder
rm -rf .git

# Connect to a new remote repository
git remote add origin https://github.com/Astra726/WJNote.git
git remote -v

# Check Git Status, to see which files have been modified or added
git status

# Initialize Git
git init

# Add your files to the Git staging area
git add .

# Commit your changes
git commit -m "Initial commit"   

# Push the changes to GitHub
git push -u origin main

Collaborating Across Multiple Computers

On any other computer, you (or collaborators) can clone the repository and continue editing:

# Clone the repository:
`git clone https://github.com/your-username/your-repository.git`

# Pull changes on other machines before working:
`git pull`
